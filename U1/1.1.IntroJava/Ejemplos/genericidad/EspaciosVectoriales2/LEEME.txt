En esta versión, la interfaz GeometriaEuclidea se encarga de implementar un metodo default que se encarga de calcular el maximo entre el objeto actual y el objeto del parámetro, de modo que ya está incorporado a cualquier objeto de una clase que implemente dicha interfaz.

Adicionalmente, las clases Coordenada2D y Coordenada3D son válidas para vectores cuyas componentes son objetos que extiendan la clase abstracta Number como por ejemplo Double, Float, Integer, Short, Long, ... Esto se hace porque es necesario obtener representaciones en coma flotante para calcular la norma vectorial (ya sea el objeto original un Float, un Double, un Integer, un Long, ...); dicha representación se obtiene mediante el empleo del método .doubleValue() (implementado convenientemente en cada clase hija: Double, Float, Integer, Long, ...). Obsérvese cómo en los ficheros test\Test2D.java y test\Test3D.java se reutiliza el mismo software tanto para Integer como para Doubles
